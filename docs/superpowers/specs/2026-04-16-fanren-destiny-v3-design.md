# 凡人修仙传·命运模拟 v3 设计文档

**日期：** 2026-04-16
**版本：** v3.0
**范围：** 动态题目系统——每道内容题根据前序答案和累积分数动态切换版本，题干和选项同时变化

---

## 1. 设计目标

让每次游玩体验都有差异。用户选择不同路径时，看到的不只是不同的题目顺序，而是不同的题目内容——题干措辞、情境描述、选项文字都随前序状态变化。

---

## 2. 动态化范围

| 类型 | 题目 | 处理方式 |
|------|------|---------|
| 动态化内容题 | Q1-Q4, QA1-QA3, QAA1-4, QAB1-4, QB1-QB3, QBA1-4, QBB1-4 | 每题2-3个版本，运行时选择 |
| 固定分叉题 | Q5, QAX, QBX | 不动态化，保持路径选择逻辑清晰 |
| 固定收尾题 | QF1-QF3 | 不动态化，保持仪式感 |

共32道题动态化。

## 3. 数据结构

### 3.1 版本条件类型

```typescript
// 条件类型
type VersionCondition =
  | { type: "prevAnswer"; value: string }           // 上一题选了某个选项文字
  | { type: "score"; dimension: keyof DestinyScores; gte: number }  // 某维度 >= 阈值
  | { type: "score"; dimension: keyof DestinyScores; lt: number }   // 某维度 < 阈值
  | { type: "default" }                             // 兜底版本，必须存在

// 每个版本
interface QuestionVersion {
  condition: VersionCondition;
  text: string;
  // slider 题专用
  leftLabel?: string;
  rightLabel?: string;
  scoring?: SliderScoring;
  // choice/image-choice 题专用
  options?: ChoiceOption[];
  // ranking 题专用（选项文字可变，维度不变）
  rankingTexts?: string[];
}

// 动态题目
interface DynamicQuestion {
  id: string;
  type: "choice" | "image-choice" | "slider" | "ranking";
  timed?: number;
  versions: QuestionVersion[];  // 按优先级排列，第一个满足条件的生效
}
```

### 3.2 条件优先级规则

运行时按顺序检查每个版本的条件，取第一个满足的：

1. `prevAnswer`：上一题的选项文字精确匹配（最高优先级）
2. `score`：累积分数阈值匹配
3. `default`：兜底（必须是最后一个版本）

## 4. 运行时选题逻辑

```typescript
function resolveQuestion(
  question: DynamicQuestion,
  prevAnswer: string | null,
  scores: DestinyScores
): ResolvedQuestion {
  for (const version of question.versions) {
    const cond = version.condition;
    if (cond.type === "prevAnswer" && prevAnswer === cond.value) return version;
    if (cond.type === "score") {
      const val = scores[cond.dimension];
      if ("gte" in cond && val >= cond.gte) return version;
      if ("lt" in cond && val < cond.lt) return version;
    }
    if (cond.type === "default") return version;
  }
  return question.versions[question.versions.length - 1]; // 安全兜底
}
```

DestinyQuiz 在渲染每道题前调用此函数，得到当前应显示的版本。

## 7. 实现说明

### 7.1 questions.json 迁移规则

- 固定题（Q5/QAX/QBX/QF1-3）：保持现有结构不变
- 动态题：改为 `versions` 数组，每个版本包含 `condition` + 完整题目内容
- 每题必须有且只有一个 `{ "type": "default" }` 版本，放在 versions 最后

### 7.2 prevAnswer 的匹配规则

- 精确匹配选项的 `text` 字段
- 上一题指"当前题目在 sequence 中的前一道题"，跨题型均适用
- 分叉题（Q5/QAX/QBX）的答案也参与 prevAnswer 匹配

### 7.3 分数条件的判断时机

- 在渲染当前题目前，使用**当前累积分数**（不含本题得分）判断
- 多个 score 条件时，按 versions 数组顺序，第一个满足的生效

### 7.4 ranking 题的动态化

- 只改 `rankingTexts`（选项文字），维度映射（dimension）保持不变
- 确保同一题的不同版本维度顺序一致，避免计分混乱
